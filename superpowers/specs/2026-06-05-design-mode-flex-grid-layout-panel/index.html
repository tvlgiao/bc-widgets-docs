    Design Mode — LAYOUT Section (Flex / Grid config)¶

    Date: 2026-06-05 Component: AI HTML widget → chatbot visual editor ("Design Mode") → Design panel Branch: feat/design-mode-structural-actions (PR #35) Status: Design spec — ready for implementation (bc-dev) Audience: bc-dev (paste-ready SVGs + markup/CSS guidance, matched to the existing panel)

    Summary¶

    Add a new LAYOUT section to the Design panel that appears only when the selected element is a flex or grid container (computed display is flex, inline-flex, grid, or inline-grid). It lets a merchant configure, with Figma-style icon controls:

    • flex-direction (flex only) — row / column / row-reverse / column-reverse
    • flex-wrap (flex only) — nowrap / wrap / wrap-reverse
    • gap (flex + grid) — row-gap + column-gap, numeric + unit
    • justify-content (flex only) — flex-start / center / flex-end / space-between
    • align-items (flex only) — flex-start / center / flex-end / stretch

    This mirrors Figma's "Auto layout" panel (flow direction icon row, gap with ↔ / ↕ affordances, alignment) but is rendered with the panel's own --ve-* tokens and existing component patterns so it is visually indistinguishable from the SPACING / SIZE / COLOR / TYPOGRAPHY / BORDER / ALIGNMENT sections already shipping.

    The deliverable is design only: markup structure, control layout, all icon SVGs, and states. No component code is committed here — bc-dev wires it.

    Goals¶

    • Surface flex/grid container layout properties in Design Mode, which today has no way to edit them.
    • Visual parity with the existing panel: same section title style, same icon-row pattern as text-align ([data-a]) / vertical-align ([data-va]), same numeric+unit input as padding (numField / [data-p] + [data-u]).
    • Aesthetic, self-explanatory icons in the established 16×16 / currentColor / stroke-1.5 idiom (no hard-coded hex; inherits the button color token).
    • Breakpoint-aware and accessible, like the rest of the panel.

    Non-goals (YAGNI for v1)¶

    • No grid-template editing (track sizing, grid-template-columns/rows, areas, auto-flow). Grid containers get gap only in v1. Note as a follow-up.
    • No align-content / justify-items / place-* shorthands.
    • No per-child controls (align-self, order, flex-grow/shrink/basis). This section edits the container, never its children.
    • No display switcher — the section reflects the element's existing display; it does not let the merchant turn flex/grid on or off.
    • No gap-shorthand collapse logic; row-gap and column-gap are edited independently (consistent with how padding edits each side independently).

    Conditional visibility¶

    Display detection¶

    The panel's existing data-ve-show mechanism gates on element category (img / media / text), computed in show() from the tag name. LAYOUT needs a different axis — computed display — so it gets its own attribute namespace rather than overloading data-ve-show.

    Define a parallel layout-mode classifier, run inside show() after the existing category loop:

    const cs = getComputedStyle(el);
const disp = cs.display;
const layoutMode =
    (disp === 'flex'  || disp === 'inline-flex') ? 'flex'  :
    (disp === 'grid'  || disp === 'inline-grid') ? 'grid'  :
    'none';

    Then drive visibility with a data-ve-layout attribute, analogous to the existing data-ve-show loop:

    root.querySelectorAll('[data-ve-layout]').forEach(node => {
    const allow = node.dataset.veLayout.split(/\s+/);
    node.hidden = !allow.includes(layoutMode);   // 'flex' | 'grid' | 'flex grid'
});
    • data-ve-layout="flex grid" → the LAYOUT <section> wrapper and the GAP field group (shown for both modes).
    • data-ve-layout="flex" → the flex-only subgroups: flex-direction, flex-wrap, justify-content, align-items.
    • There is no grid-only subgroup in v1.

    When layoutMode === 'none', the section wrapper is data-ve-layout="flex grid" which excludes none, so the whole section is hidden — exactly the desired "only show for flex/grid" behavior. No extra branch needed.

    Re-evaluation on breakpoint change¶

    display can itself differ per breakpoint (an element may be flex on desktop and block on mobile via a @media override). setActiveBp() → refresh() already re-reads the cascade. The layout-mode classification must therefore be recomputed inside refresh() (doRefresh), not only in show(), so switching the Desktop/Tablet/Mobile tabs correctly shows/hides the section. Recommendation: extract the data-ve-layout visibility loop into a small applyLayoutVisibility(el) helper called from both show() and doRefresh().

    Edge note: read display from the cascade-at-activeBp where available (computeValue(index, el, 'display', activeBp)), falling back to getComputedStyle().display, mirroring how the rest of doRefresh resolves values. This keeps the section in sync with the breakpoint being previewed rather than the live viewport.

    Control layout¶

    The section sits after BORDER and before ALIGNMENT in [data-ve-edit] (structure → spacing → size → color → type → border → layout → alignment). LAYOUT is container-level and pairs naturally with ALIGNMENT (which is text-level), so they read as a related pair at the bottom of the panel.

    Section shell matches every other section exactly:

    <div class="ai-html-chatbot__ve-sec" data-ve-layout="flex grid">
    <h4 class="ai-html-chatbot__ve-sec-title">Layout</h4>
    ...
</div>
    • Title: Layout in the ve-sec-title style (already uppercased + tracked via CSS text-transform: uppercase; letter-spacing: 0.1em — write it title-case in markup like the others, e.g. Spacing, Border).
    • Sub-labels (direction, wrap, row, column, justify, align) use the existing .ai-html-chatbot__ve-field label style (11px, --ve-text-2, lowercase as written), same as left / right / text-align.
    • Separators between sub-groups use the existing <div class="ai-html-chatbot__ve-gap"></div>.

    Data-attribute namespaces (new)¶

    Group Attr Values CSS property
    flex-direction data-fd row column row-reverse column-reverse flex-direction
    flex-wrap data-fw nowrap wrap wrap-reverse flex-wrap
    justify-content data-jc flex-start center flex-end space-between justify-content
    align-items data-ai flex-start center flex-end stretch align-items
    row gap data-p="row-gap" + data-u="row-gap" numeric + unit row-gap
    column gap data-p="column-gap" + data-u="column-gap" numeric + unit column-gap

    These four icon groups are wired exactly like the existing [data-a] / [data-va] / [data-bs] groups in panel.js: a click loops the group, toggles .active onto the clicked button, calls deps.onEdit(selected, elementClass(selected), <css-prop>, btn.dataset.<x>, activeBp), then updateResetVisibility(). The two gap fields plug into the existing [data-p] / [data-u] machinery with zero new wiring once row-gap and column-gap are added to PX_CONTROLS (see below).

    panel.js / resolver touch-points (for bc-dev, not committed here)¶

    • Add 'row-gap', 'column-gap' to the PX_CONTROLS set so the gap fields use the numeric+unit parse/refresh path and get reset-button tracking for free.
    • In doRefresh(), add active-state sync for [data-fd] / [data-fw] / [data-jc] / [data-ai] mirroring the [data-bs] block: read the cascade-at-bp value (fallback computed style), lowercase it, toggle .active.
    • flex-direction: default row.
    • flex-wrap: default nowrap.
    • justify-content: computed often returns normal → treat normal/empty as "no selection" (no .active button), matching how vertical-align buckets to ''.
    • align-items: computed default for flex is normal (renders as stretch). Treat normal as stretch for the active highlight so the panel reflects real behavior; but do not auto-write stretch — only stamp it on click.
    • Add the four new props to updateResetVisibility() (the [data-reset-p] lookups), same pattern as text-align / vertical-align.
    • Add applyLayoutVisibility(el) and call from show() and doRefresh().

    Aesthetic SVG icons¶

    All icons: 16×16 viewBox, stroke="currentColor", fill="none" (stroke-based), stroke-width="1.5", stroke-linecap="round", stroke-linejoin="round". Rendered at width="14" height="14" inside the ve-align-row buttons to match the text-align icons. No hard-coded color — they inherit the button's --ve-text-2 (idle) / --ve-accent (active) via currentColor, exactly like [data-a]. Filled accents (the "moving" item in direction icons, the packed items in justify/align) use fill="currentColor" on a single small shape so the icon reads instantly while still inheriting the token.

    Each constant below is paste-ready. Name them to match the existing ICON_* convention in panel-template.js.

    Small 12×12 directional glyphs that sit before the row / column labels in the gap field label-row (Figma uses ↕ / ↔ here). Stroke-based, currentColor, inherit the label's --ve-text-2. Two bars with a double-headed arrow showing the spacing axis: vertical arrow between two horizontal bars (row gap), horizontal arrow between two vertical bars (column gap).

    States¶

    All four icon groups reuse the existing .ai-html-chatbot__ve-align-row component verbatim — no new CSS class is required for the buttons themselves, so they inherit idle / hover / active styling that already ships and is tested with text-align / vertical-align:

    State Styling (already defined for ve-align-row button)
    Idle color: var(--ve-text-2), transparent bg
    Hover (not active) background: var(--ve-surface-2), color: var(--ve-text)
    Active / selected background: var(--ve-accent-bg), color: var(--ve-accent), box-shadow: 0 0 0 1px var(--ve-accent-border)
    Disabled n/a for these groups in v1 (no per-value disable)

    Focus-visible (a11y addition). The existing ve-align-row button has no focus ring. Add one scoped rule so keyboard users see focus on every icon group (text-align/vertical-align benefit too). Use the same ring the breadcrumb uses:

    outline: 2px solid var(--ve-accent);
    outline-offset: -1px;
}

    (outline-offset: -1px keeps the ring inside the 2px-padded row so it does not clip against the row border.)

    Responsive / breakpoint behavior¶

    • The section is fully breakpoint-aware like every other section: edits land in the active breakpoint's @media block via deps.onEdit(..., activeBp), and refresh() re-reads cascade-at-bp on tab switch.
    • Critically, the section's own visibility is breakpoint-dependent because display can change per breakpoint — so the data-ve-layout visibility loop must run inside doRefresh() (see "Re-evaluation on breakpoint change").
    • The panel itself is a fixed-width sidebar; the two-up ve-grid2 rows collapse gracefully at the panel's own narrow width via the existing grid rules. No new media queries needed inside the panel.

    Accessibility¶

    • Every icon button is <button type="button"> with a human-readable aria-label (the CSS value in plain words: "Row", "Column", "Space between", "Stretch", etc.) plus a matching title for mouse tooltips. Icons are decorative (aria-hidden is implied; the SVGs carry no text), so the accessible name comes from aria-label.
    • Each icon group is wrapped with role="radiogroup" and an aria-label naming the property ("Flex direction", "Justify content", …). The buttons behave as a single-select / radio-like group: clicking one activates it and clears the others (already the behavior of the [data-a]/[data-va]/[data-bs] click loop — replicate it for the new attrs).
    • Optional polish (recommend if cheap): reflect selection with aria-checked="true|false" on each button and role="radio", so screen readers announce the active option. If bc-dev keeps it to radiogroup + .active only, that is acceptable for v1 parity with the existing groups.
    • Color is not the sole signal of the active state: the active button also gets the box-shadow ring (1px accent border), so the selection is perceivable without relying on the accent fill alone (WCAG 1.4.1 Use of Color).
    • Contrast: --ve-accent (#4f46e5) on --ve-accent-bg (#eef2ff) and --ve-text-2 (#475569) on --ve-surface (#ffffff) both clear 4.5:1, matching the rest of the panel.
    • Gap glyphs are aria-hidden="true" (the adjacent label text "row" / "column" is the accessible name).
    • Focus order follows DOM order: direction → wrap → row-gap → column-gap → justify → align, which reads logically top-to-bottom.

    Future follow-ups (noted, out of scope for v1)¶

    • Grid track editing: grid-template-columns / grid-template-rows (track-count + size), grid-auto-flow, named areas. This is the big one — it needs its own control paradigm (track list editor) and is deliberately deferred; grid containers get gap-only now.
    • align-content / justify-items / place-content for multi-line flex and grid.
    • Per-child controls (align-self, order, flex grow/shrink/basis) exposed when a flex/grid item (not container) is selected.
    • A display switcher to convert an element to/from flex/grid.
